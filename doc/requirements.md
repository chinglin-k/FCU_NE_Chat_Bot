# 逢甲大學福星宿舍網路報修 Chatbot — 需求規格書

**版本**：1.0  
**建立日期**：2026-07-17  
**專案倉庫**：https://github.com/chinglin-k/FCU_NE_Chat_Bot

---

## 1. 專案背景

@chinglin-k 需要一套自動化工具，協助學生快速取得網路相關教學資源，並能自助發起報修通報，減少處理重複性問題的時間。

---

## 2. 系統角色

| 角色 | 說明 |
|---|---|
| **學生（使用者）** | 透過 Chatbot 自助查詢教學文件、查詢常見問題，或發起報修通報 |
| **管理員（網管人員）** | 接收 Google 試算表中的通報案件，進行派人維修與案件追蹤 |

---

## 3. 功能需求

### 3.1 選項按鈕介面

Chatbot 啟動後顯示三個主要選項按鈕：

| 按鈕 | 觸發行為 |
|---|---|
| 📚 教學 | 顯示 Windows / Mac 系統選擇，提供 PDF 教學文件連結 |
| ⚙️ 常見問題 | 常見問題一覽：WiFi 帳號密碼、轉接器馨動程式、寢室 WiFi 訊號、冷氣電費儲値等 |
| 🔧 我要實體協助、報修 | 開啟報修表單 |

### 3.2 LLM 語意分析與意圖分類

使用者輸入文字時，透過 Gemini API（由 Google Apps Script 代理）分析語意，分類為以下意圖：

| 意圖代碼 | 判斷條件 | 回覆行為 |
|---|---|---|
| `BUTTON_TEACH` | 詢問教學、設定方法 | 提供系統選擇按鈕 |
| `BUTTON_SETTING` | 詢問轉接器、驅動程式問題 | 回覆韌體建議 |
| `BUTTON_REPORT` | 明確要求報修 | 開啟報修表單 |
| `STICKER_PORT` | IP 貼紙缺漏或網路孔故障 | 開啟報修表單 |
| `NON_NETWORK` | 非網管業務（冷氣、洗手台等） | 轉介宿舍服務台或行動逢甲 |
| `UNKNOWN` | 無法判斷 | 回覆無法提供答案 |

### 3.3 教學文件回覆

- **Windows 教學**：提供 Google Drive PDF 連結
- **Mac 教學**：提供 Google Drive PDF 連結
- WiFi 分享器設定：建議依照機種說明書操作

### 3.4 報修（通報）模組

收集以下資料：

| 欄位 | 說明 | 備註 |
|---|---|---|
| 姓名 | 學生姓名 | 必填 |
| 學號 | 學生學號 | 必填 |
| 房號 | 宿舍房號（例：A123） | 必填 |
| 床號 | 床位號碼（例：1） | 必填 |
| 手機 | 聯絡電話 | 必填 |
| 可維修時間 | 本人需在場 | 必填；輸入框下方純文字提示：18:00–21:00（新生入住期間 12:00–17:00 另外說明） |
| 問題描述 | 網路或設備問題說明 | 必填 |
| 通報日期時間 | 系統自動填入 | 自動，不顯示給使用者 |

寫入 Google 試算表欄位：日期、時間、學號、姓名、房號、床號、手機、可維修時間、問題描述、是否派人（留空）、是否完成（留空）、備註（留空）

### 3.5 累積使用人數統計

- 每次新 session 開啟時累加一次
- 計數儲存於 GAS Script Properties
- 在 Header 顯示「已協助 X 人 ✨」

### 3.6 部署

- 操作介面部署至 **GitHub Pages**（靜態前端）
- 後端透過 **Google Apps Script Web App** 提供服務

---

## 4. 非功能需求

| 項目 | 要求 |
|---|---|
| 安全性 | Gemini API Key 僅存於 GAS Script Properties，不得出現在前端程式碼或 Git |
| 效能 | LLM 分類時顯示打字指示器，提升等待體驗 |
| 相容性 | 支援桌機與手機瀏覽器，行動裝置優化 |
| 語言 | 繁體中文介面 |
| 無障礙 | 使用語意化 HTML、aria 屬性 |

---

## 5. 系統限制

- 前端為靜態頁面（GitHub Pages），無伺服器端邏輯
- API Key 透過 GAS 代理隱藏，前端不持有任何機密資訊
- 可維修時間欄位**不做時間格式驗證**，僅以純文字提示使用者填寫範圍
